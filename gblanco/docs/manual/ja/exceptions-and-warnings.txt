++ マネージャの例外

接続管理で何かが失敗した場合Doctrine_Manager_Exceptionがスローされます

<code type="php">
try {
    $manager->getConnection('unknown');
} catch (Doctrine_Manager_Exception) {
    // エラーをキャッチする
}
</code>

++ リレーションの例外

リレーションの解析中に何かが失敗した場合リレーションの例外がスローされます。

++ 接続の例外

データベースレベルで何かが失敗した場合接続の例外がスローされます。Doctrineは完全にポータブルなデータベースエラーハンドリングを提供します。このことが意味するのはsqliteもしくは別のデータベースを使おうが常に発生したエラーに対してポータブルなエラーコードとメッセージを得ることができます。

<code type="php">
try {
    $conn->execute('SELECT * FROM unknowntable');
} catch (Doctrine_Connection_Exception $e) {
    print 'Code : ' . $e->getPortableCode();
    print 'Message : ' . $e->getPortableMessage();
}
</code>
